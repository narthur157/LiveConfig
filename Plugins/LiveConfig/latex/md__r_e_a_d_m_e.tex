 Treat your game data with respect.

Live Config allows you to decouple tuning concerns from feature implementation. Create tuning values on the fly while coding, and empower designers to refine them in-\/editor or in-\/build without further engineering support.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Why use Live Config?}{Why use Live Config?}}\label{md__r_e_a_d_m_e_autotoc_md1}
Tuning and gameplay iteration shouldn\textquotesingle{}t be an afterthought buried in Blueprints or hardcoded constants. Live Config pulls your tuning data away from the implementation, giving it the respect it deserves.


\begin{DoxyItemize}
\item {\bfseries{Stop Thinking About Tuning While Coding}}\+: Create an initial property, use it in your code, and move on.
\item {\bfseries{Empower Designers}}\+: Designers can iterate on values in the editor or live in-\/build via Google Sheets.
\item {\bfseries{SCM-\/\+First Workflow}}\+: Changes are stored in individual JSON files, making tuning history as readable and manageable as code.
\item {\bfseries{Big Studio Tooling for Everyone}}\+: Smaller teams often lack the resources to build dedicated tools teams. Live Config provides the robust configuration infrastructure used by AAA studios, out of the box.
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysection{\texorpdfstring{Key Features}{Key Features}}\label{md__r_e_a_d_m_e_autotoc_md2}

\begin{DoxyItemize}
\item {\bfseries{Editor Tooling}}\+:
\begin{DoxyItemize}
\item {\bfseries{Live Config Manager}}\+: A dedicated window to manage all your properties, tags, and overrides.
\item {\bfseries{Blueprint Integration}}\+: Special blueprint node for easy access to config properties, provides type-\/safe access and autocomplete.
\item {\bfseries{CVar per-\/property}}\+: Automatic autocomplete for live config properties in the console. Easily test editing values
\end{DoxyItemize}
\item {\bfseries{Built for source control}} Live config uses one JSON file per property, preventing excessive merge conflicts and creating a readable diff for tuning changes.
\item {\bfseries{Exportable game database}} Easily create a wiki or other knowledge store using your game\textquotesingle{}s actual data
\item {\bfseries{Property Tags}} Tagging properties provides opportunities not only for organizing, but building additional tooling around properties
\item {\bfseries{Google Sheets Integration}}\+: Use a public readable Google Sheet (published as CSV) to override config values at runtime.
\item {\bfseries{Type support}}\+: Properties are tied to particular types, including structs.
\item {\bfseries{Curve Table Support}}\+: Seamlessly sync remote values with Unreal\textquotesingle{}s native Curve Tables. This is particularly useful for {\ttfamily FScalable\+Float} from GAS
\item {\bfseries{Profile System}}\+: Create sets of tuning changes and save them to their own file.
\item {\bfseries{Network Replication}}\+: Server-\/authoritative config values are efficiently replicated to clients.
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysection{\texorpdfstring{Comparison with Similar Tools}{Comparison with Similar Tools}}\label{md__r_e_a_d_m_e_autotoc_md3}

\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{\texorpdfstring{1. Basic Setup}{1. Basic Setup}}\label{md__r_e_a_d_m_e_autotoc_md5}

\begin{DoxyEnumerate}
\item Enable the {\bfseries{Live Config}} plugin in your project.
\item In the Unreal Editor, go to {\bfseries{Project Settings}} -\/\texorpdfstring{$>$}{>} {\bfseries{Project}} -\/\texorpdfstring{$>$}{>} {\bfseries{Live Config}}.
\item Set your {\bfseries{Sheet URL}}. This should be a link to a Google Sheet published as CSV.
\begin{DoxyItemize}
\item {\itshape Google Sheets\+: File -\/\texorpdfstring{$>$}{>} Share -\/\texorpdfstring{$>$}{>} Publish to web -\/\texorpdfstring{$>$}{>} Select Sheet and \textquotesingle{}Comma-\/separated values (.csv)\textquotesingle{} -\/\texorpdfstring{$>$}{>} Copy Link.}
\end{DoxyItemize}
\end{DoxyEnumerate}\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysubsection{\texorpdfstring{2. Defining Properties}{2. Defining Properties}}\label{md__r_e_a_d_m_e_autotoc_md6}
Open the {\bfseries{Live Config Property Manager}} from the {\bfseries{Live Config}} button in the Level Editor toolbar (next to the Play button).

From here you can\+:
\begin{DoxyItemize}
\item Add new properties.
\item Assign tags (e.\+g., {\ttfamily Combat}, {\ttfamily Movement}).
\item Set default values and types.
\item Import/\+Export from Curve Tables.
